## 工作原则

* README.md中包含了项目的需求，并记录尚未解决的Bug或待开发的需求
* 有任何项目的修改，你要及时更新README.md
* 当用户给你下达开发新功能的指令后，你要把你的想法和用户确认清楚再开发
* 你编写完功能后，必须进行详细的测试
* 开发新功能或者修改功能后，你要及时修改example/目录下的示例

## 开发流程

[用户提出宽泛的需求] -> [你进行引导式地细化需求] -> [用户确认需求] -> [你制定开发计划] -> [用户同意] -> [你更新README.md] -> [你编写代码] -> [你测试代码] -> [用户验收确认] -> [你完善README.md] -> [你提交代码]
